# Security Policy

## Reporting

If you find a security issue, do not open a public GitHub issue first.

Send a private report to the repository owner through GitHub security advisory
or email, and include:

- affected version
- reproduction steps
- potential impact
- any suggested mitigation

## Scope

The highest-priority issues are:

- command execution or shell injection
- unsafe file writes
- exposed credentials or tokens
- insecure API defaults when running `llmfit serve`

We will acknowledge valid reports and work on a fix before public disclosure.
